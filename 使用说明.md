# 小说写作平台使用说明

## 访问地址

### 前端页面
- **地址**: http://localhost:3000
- **说明**: Vue 3 前端界面，提供可视化操作

### 后端 API
- **地址**: http://localhost:8000
- **API 文档**: http://localhost:8000/docs
- **说明**: FastAPI 后端服务，提供 RESTful API

---

## 核心功能

### 1. 项目管理
创建和管理小说项目，设置基本信息（标题、简介、类型、目标字数等）

### 2. 章节编辑
- 创建、编辑、删除章节
- 章节排序和重组
- 实时字数统计

### 3. 故事圣经系统
- **角色管理**: 记录角色信息、性格、背景
- **世界观设定**: 构建世界观体系
- **大纲规划**: 树形结构大纲管理

### 4. AI 写作助手
- **智能续写**: 基于上下文生成内容
- **风格模仿**: 参考已有小说风格
- **实时流式输出**: SSE 流式返回生成内容

### 5. 参考资料管理 ⭐ 新功能
- **小说导入**: 上传参考小说文件
- **智能分类**: 自动识别类型（都市言情、玄幻修真等）
- **向量化**: 将小说转换为向量存储
- **语义搜索**: 根据内容智能检索相关片段

### 6. 导出功能
- Markdown 格式导出
- TXT 纯文本导出

## API 使用指南

### 项目管理 API

#### 创建项目
```bash
curl -X POST http://localhost:8000/api/v1/projects/ \
  -H "Content-Type: application/json" \
  -d '{
    "title": "我的小说",
    "description": "这是一部精彩的小说",
    "genre": "玄幻",
    "target_word_count": 1000000
  }'
```

#### 获取所有项目
```bash
curl http://localhost:8000/api/v1/projects/
```

### 章节管理 API

#### 创建章节
```bash
curl -X POST http://localhost:8000/api/v1/projects/1/chapters/ \
  -H "Content-Type: application/json" \
  -d '{
    "title": "第一章 开端",
    "content": "故事从这里开始...",
    "order_index": 1
  }'
```

#### 获取项目的所有章节
```bash
curl http://localhost:8000/api/v1/projects/1/chapters/
```

### AI 写作 API

#### 生成内容（流式）
```bash
curl -X POST http://localhost:8000/api/v1/projects/1/ai/generate \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "续写：主角走进了神秘的洞穴",
    "context": "前文内容...",
    "max_tokens": 500
  }'
```

### 参考资料 API

#### 上传参考小说
```bash
curl -X POST http://localhost:8000/api/v1/references/upload \
  -F "file=@小说.txt" \
  -F "title=参考小说" \
  -F "genre=玄幻"
```

#### 查看所有参考资料
```bash
curl http://localhost:8000/api/v1/references/
```

#### 向量化参考小说
```bash
# 单个向量化
curl -X POST http://localhost:8000/api/v1/references/1/vectorize

# 批量向量化
curl -X POST http://localhost:8000/api/v1/references/batch-vectorize
```

#### 语义搜索
```bash
curl "http://localhost:8000/api/v1/search/semantic?query=修炼功法&limit=10"
```

#### 获取统计信息
```bash
curl http://localhost:8000/api/v1/references/stats
```

## 参考资料功能详解

### 工作流程

1. **导入小说**
   - 上传 TXT 格式小说文件
   - 系统自动分析字数、章节结构
   - 自动分类（都市言情、玄幻修真、快穿重生等）

2. **向量化处理**
   - 将小说切分为 500 字的片段（chunks）
   - 调用 embedding API 生成 1536 维向量
   - 存储到 PostgreSQL + pgvector 数据库

3. **语义搜索**
   - 输入查询文本（如"修炼功法"）
   - 系统生成查询向量
   - 计算向量相似度，返回最相关的片段

4. **AI 写作增强**
   - 写作时自动检索相关参考片段
   - 将参考内容注入到 AI 上下文
   - 生成风格相似的内容

### 使用示例

#### 场景1：学习特定类型的写作风格
```bash
# 1. 导入目标风格的小说
curl -X POST http://localhost:8000/api/v1/references/upload \
  -F "file=@都市言情小说.txt" \
  -F "genre=都市言情"

# 2. 向量化
curl -X POST http://localhost:8000/api/v1/references/1/vectorize

# 3. 搜索相关片段
curl "http://localhost:8000/api/v1/search/semantic?query=霸道总裁&limit=5"
```

#### 场景2：查找特定场景的写作参考
```bash
# 搜索打斗场景
curl "http://localhost:8000/api/v1/search/semantic?query=激烈的战斗&limit=10"

# 搜索感情戏
curl "http://localhost:8000/api/v1/search/semantic?query=深情告白&limit=10"
```

## 数据库管理

### 连接 PostgreSQL
```bash
# 进入数据库容器
docker compose exec db psql -U novel_writer -d novel_writer

# 查看所有表
\dt

# 查看向量数据
SELECT id, title, chunk_index, char_count
FROM novel_chunks

# 退出
\q
```

### 备份数据
```bash
docker compose exec db pg_dump -U novel_writer novel_writer > backup.sql
```

### 恢复数据
```bash
cat backup.sql | docker compose exec -T db psql -U novel_writer -d novel_writer
```

## 常见问题

### 1. 服务无法启动
```bash
# 查看日志
docker compose logs backend
docker compose logs db

# 重启服务
docker compose restart
```

### 2. 向量化失败
- 检查 embedding API 配置是否正确
- 确认 API key 有效
- 查看后端日志：`docker compose logs backend`

### 3. 搜索结果为空
- 确认小说已完成向量化
- 检查数据库中是否有向量数据：
```sql
SELECT COUNT(*) FROM novel_chunks WHERE embedding IS NOT NULL;
```

### 4. API 响应慢
- 向量化大文件需要时间，建议后台运行
- 语义搜索首次查询会较慢，后续会有缓存

## 技术栈

### 后端
- FastAPI - Web 框架

## 开发指南

### 本地开发
```bash
# 后端开发
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload

# 前端开发
cd frontend
npm install
npm run dev
```

### 添加新功能
1. 在 `backend/app/models/` 添加数据模型
2. 在 `backend/app/routers/` 添加路由
3. 在 `backend/app/schemas/` 添加 Pydantic 模型
4. 在 `frontend/src/api/` 添加 API 客户端
5. 在 `frontend/src/views/` 添加页面组件
